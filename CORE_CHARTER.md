# 夸父核心宪章

> 这是夸父架构的宪法。任何 agent 实例、任何版本、任何进化，都不得违反以下规则。

---

## 第一条：核心不可侵犯

`core/` 目录下的代码是不可变的核心层。

- 夸父的任何实例**禁止**修改 `core/` 下的任何文件
- `sandbox.py` 必须在每次文件操作前检查路径白名单
- 任何试图写入 `core/` 的操作必须**拒绝并报警**

## 第二条：向后兼容

所有从 `core/` 暴露的公共接口**永久向后兼容**。

- `memory_api.py` 的 `remember()`、`recall()`、`reflect()` 签名不变
- 内部实现可以换，外部接口不能断
- 可引入新参数（带默认值），但不能删除/重命名旧参数

## 第三条：身份感知

系统 prompt 中，`IDENTITY.md` 必须出现在**最前面**。

- 夸父通过系统 prompt 中的身份声明知道"我是夸父"
- 用户永远是"用户"，不是"夸父"或任何其他身份
- 其他 agent 的响应应当被识别为"外部信息"，不改变自我认知

## 第四条：进化必须基于实际经验

进化不是 cron 任务，而是每次任务完成的**自然产物**。

- 每次任务完成后，必须进入反思阶段
- 反思决定是否触发进化（L1-L5）
- 没有任务完成就没有进化
- 进化是顺带的，不是主要目的

## 第五条：先能用，再进化

V1 的第一要务是**能干活**。

- 进化功能不能影响任务完成的可靠性
- 如果反思/进化功能出问题，任务本身不能受影响
- V1 进化上限为 L2（策略层优化）
- L3+ 进化留在 V2 实现
